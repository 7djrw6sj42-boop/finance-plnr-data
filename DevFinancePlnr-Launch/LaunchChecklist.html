PART 1 — Launch Steps (in order)

Admin Tasks (only you can do)

Step 1 — Apple Developer Program enrollment ($99/yr)

  • Go to developer.apple.com/programs/enroll
  • Sign in with Apple ID tied to your real identity
  • Choose Individual (Organization needs DUNS number)
  • Pay $99/yr
  • Wait 24-48 hours for approval email

Step 2 — Google Play Console account ($25 once)

  • Go to play.google.com/console
  • Pay $25 one-time
  • Use same identity as Apple

Step 3 — Privacy Policy hosted

  • Privacy Policy live at github.io URL
  • Wired into Settings → Privacy Policy in app

Step 4 — Support page / URL

  • Create simple HTML page with contact email
  • Host on same GitHub Pages site
  • URL: https://7djrw6sj42-boop.github.io/finance-plnr-data/support.html

Step 5 — Take screenshots

  • Setup iOS simulator (iPhone 16 Pro Max) or use real phone
  • Resolution: 1290×2796
  • Min 3, max 10 screenshots
  • #1 Home dashboard
  • #2 Deal Analyzer with results
  • #3 Market Scorecard with verdict
  • #4 MAO Calculator with result
  • #5 Driving for $$$ list with photos
  • #6 Cash Buyers + Share modal
  • #7 Helper bot conversation OR Contract Generator
  • Optional: app-mockup.com for phone bezels

Build Tasks (I can run for you)

Step 6 — Set up EAS Build

  • npm install -g eas-cli
  • eas login
  • eas build:configure
  • Verify eas.json created

Step 7 — Build iOS app

  • eas build --platform ios --profile production
  • Wait 15-30 min for cloud build
  • Confirm .ipa artifact ready

Step 8 — Submit build to App Store Connect

  • eas submit --platform ios
  • OR upload manually via Transporter app
  • Build appears in App Store Connect after ~15 min

Step 9 — Build + Submit Android

  • eas build --platform android --profile production
  • eas submit --platform android

Final Stretch

Step 10 — Fill App Store Connect form

  • Use pre-fills from AppStoreFormFields.md
  • Sign in to appstoreconnect.apple.com
  • Create new app — name, subtitle, bundle ID, language
  • Pricing — $0 free, all territories
  • Version 1.0 — promotional text, description, keywords, what's new
  • Upload icon + screenshots
  • Categories — Finance / Business
  • Age rating — answer all None / No → 4+
  • Privacy nutrition label (use pre-filled table)
  • Privacy Policy URL
  • Support URL
  • Add review notes

Step 11 — TestFlight

  • App Store Connect → TestFlight tab
  • Add 5-10 internal testers
  • Send invite link
  • Beta test 1-2 weeks
  • Iterate / push update if needed

Step 12 — Submit for App Review

  • Review all metadata one final time
  • Click "Add for Review"
  • Export Compliance — encryption (HTTPS) → exemption
  • Content Rights — Yes (data sources)
  • Advertising Identifier (IDFA) — No
  • Wait 1-3 business days for Apple review

Step 13 — Approval & Release

  • Approval email received
  • Click Release (or auto-release on approval)
  • Live globally within hours

Step 14 — Google Play Production

  • Move from Internal → Closed → Open testing → Production
  • Google review usually <24 hours
  • Play Store listing live

URL / Linking

Long-Running State

PDF Export

  • PDF on PSA → opens in Files / Mail / AirDrop?
  • PDF for budget with 0 line items → renders?
  • Special chars in seller name → renders cleanly?

Settings / Wipes

  • Wipe All App Data → returns to fresh install?
